## ⚠️ 发现的问题

### 问题 1: headless 浏览器文件上传受限（技术限制）

**描述**: Gradio 的文件上传组件（`<input type="file">` 隐藏在上传按钮后）在 headless 浏览器环境中无法正常触发文件选择对话框。导致无法自动化测试「上传 CSV → 数据解析 → 训练 → 下载 ZIP」的完整端到端流程。

**影响**: 无法在自动化测试中验证完整训练→下载流程。

**建议**:
1. 手动测试：浏览器打开 `http://localhost:7861`，手动上传 `test_data/shampoo.csv` 测试完整流程
2. API 测试：使用 `gradio_client` Python 库调用 `/wiz_on_train` API
3. 或者：开发命令行测试脚本，直接调用 `src/` 核心模块

### 问题 2: 历史日志显示 v1.04 有训练失败记录

**描述**: `logs/` 目录下有旧日志显示 v1.04 时代 RandomForest 因 `seq_len` 参数导致训练失败。

**状态**: 这是旧版本（v1.04）的问题，v1.5 已重构，可能已修复。建议手动验证。
